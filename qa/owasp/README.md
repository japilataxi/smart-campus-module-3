# OWASP Dependency-Check - Vulnerability Testing

This folder stores the documentation and evidence for the vulnerability testing phase of the Smart Campus Module 3 QA work.

## 1. Objective

The objective of this phase is to identify known vulnerabilities in the project dependencies using an automated Software Composition Analysis approach. The analysis helps detect vulnerable packages, related CVEs, severity levels, and remediation recommendations before delivering the final QA evidence.

## 2. Tool

The tool used for this phase is OWASP Dependency-Check.

OWASP Dependency-Check analyzes project dependencies and compares them against public vulnerability databases such as the National Vulnerability Database. The expected output for this project is an HTML report that can be reviewed and attached as QA evidence.

## 3. Workflow Used

The GitHub Actions workflow configured for this analysis is:

```text
.github/workflows/dependency-check-prueba.yml
```

This workflow is configured to run OWASP Dependency-Check over the repository and generate an HTML report.

## 4. How to Execute the Workflow Manually

To execute the workflow manually from GitHub Actions:

1. Open the repository in GitHub.
2. Go to the `Actions` tab.
3. Select the workflow named `Auditoria de Seguridad - Dependency-Check`.
4. Click `Run workflow`.
5. Select the branch to analyze.
6. Confirm the execution.
7. Wait until the workflow finishes.

## 5. Where to Download the Generated Artifact

After the workflow finishes:

1. Open the completed workflow run.
2. Scroll to the `Artifacts` section.
3. Download the artifact named similar to:

```text
reporte-seguridad-owasp-<run_number>
```

The artifact contains the OWASP Dependency-Check report generated by the workflow.

## 6. Where to Store the Downloaded HTML Report

The downloaded HTML report must be stored in:

```text
qa/owasp/reports/
```

This folder is reserved for official OWASP Dependency-Check reports used as final QA evidence.

## 7. Screenshots to Store

Screenshots related to the vulnerability testing phase must be stored in:

```text
qa/owasp/screenshots/
```

Recommended screenshots:

- GitHub Actions workflow execution screen.
- Successful workflow completion status.
- Artifact download section.
- OWASP Dependency-Check HTML report summary.
- Vulnerability severity summary.
- Main CVE details if relevant.

## 8. Results to Summarize Later

The summary must be created only after a new OWASP Dependency-Check report exists. The following results should be extracted from the report:

- Total dependencies analyzed.
- Critical vulnerabilities.
- High vulnerabilities.
- Medium vulnerabilities.
- Low vulnerabilities.
- Main CVEs detected.
- Technical recommendations.

The future summary should be stored in:

```text
qa/owasp/summary/
```
